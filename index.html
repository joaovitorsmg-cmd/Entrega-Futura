Configuração

Sincronização com a nuvem (JSONBin) e fallback automático direto no GitHub quando o histórico fica grande demais para o plano grátis.

Sincronização automática com fallback para o GitHub: a cada importação, o painel tenta publicar o histórico completo direto na nuvem (JSONBin). Quando cabe no limite do plano grátis (~100KB), todo mundo vê na hora. Quando o histórico fica grande demais: o painel grava o arquivo direto na branch principal do GitHub, sem criar branch nem Pull Request. Sem Token do GitHub configurado, o painel só avisa que não conseguiu sincronizar; ele nunca perde os dados desta sessão.

Dados e base

Configuração

Guardada apenas neste navegador (localStorage). Nunca sai do seu computador.

Use um PAT fine-grained restrito a este repositório, com permissão de Contents: Read and write + Pull requests: Read and write. O token só fica no navegador onde for configurado, nunca é publicado.

Estado e verificação

Indicadores do selo no topo:

  • Sincronizado — dados gravados direto na nuvem.
  • Conectado · nuvem vazia — conexão ok, nenhuma importação registrada ainda.
  • Publicado direto no GitHub — histórico grande demais para o JSONBin; o arquivo foi gravado direto na branch principal do repositório.
  • Erro de sync — não foi possível gravar em nenhum dos dois destinos.
  • Sem config — Master Key/Bin ID não configurados.
