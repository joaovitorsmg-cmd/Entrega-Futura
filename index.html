Como ler — regra de cobrança

Cobrar total   título quitado: o cliente já pagou por todo o saldo parado. Force a entrega integral.

Cobrar parcial   parte do saldo já paga: libere a entrega só da quantidade paga; o restante fica retido até quitar.

Retido   cliente ainda não pagou: não entregue (risco). Se enrolar, o caminho é a devolução da entrega futura.

Configuração

Sincronização exclusiva via GitHub. Histórico, cobranças e base atual gravados direto no repositório — sem dependências externas.

Sincronização exclusiva via GitHub: a cada importação, o painel grava o histórico completo (semanas, cobranças, meses) e a base atual diretamente na branch principal do repositório — sem serviços externos. Qualquer usuário que abrir o link verá a posição mais recente sem precisar importar o arquivo. A leitura é pública (sem token); a escrita exige um PAT fine-grained com Contents: Read and write.

Importar novo relatório

Carrega um novo arquivo XLS e descarta a posição atual exibida no painel. Cobranças e histórico gravados no GitHub não são apagados.

Configuração GitHub

Gere em GitHub → Settings → Developer settings → Fine-grained tokens. Restrinja ao repositório abaixo e marque Contents: Read and write. O token fica só neste navegador, nunca é publicado.

Estado e verificação

Indicadores do selo no topo:

  • GitHub · sincronizado — histórico e cobranças gravados no repositório.
  • GitHub · sem histórico — repositório acessível, nenhuma importação registrada ainda.
  • Erro de sync — falha ao gravar. Verifique token e permissões.
  • Sem config — token ou repositório não configurados.
